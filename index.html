About Go

The Why - The How - The What

  • Designed at Google in 2007
  • Presented 2009 within a Google Tech Talk
  • v1.0 released in 2012
  • Current stable release v1.23

Motivation

  • Its designers were primarily motivated by their shared dislike of C++

Design Goals

  • Static typing and run-time efficiency (like C)
  • Readability and usability (like Python)
  • High-performance networking and multiprocessing

Go Language Design

What we've got

  • statically typed
  • strongly typed
  • compiled (fast compile time)
  • built-in concurrency
  • garbage collection
  • powerful standard library
  • built-in testing library
  • built-in profiler
  • built-in benchmarking
  • tooling 
    go mod, go vet, go fmt

Arrays 
                    
                        // An Array is just a fixed-size list of elements
                        var list [3]int         // array of length 3
                        list[0] = 1             // assign value to index 0
                        fmt.Println(list)       // => [1 0 0]

                        list := [3]int{1, 2, 3} // initialized with 3 elements
                        fmt.Println(len(list))  // => 3 (len -> length)
Slices 
                    
                        // A Slice is a special representation
                        // of an array with a dynamic size
                        var slice []int              // empty slice
                        slice := make([]int)         // empty slice
                        slice := make([]int, 0, 100) // empty slice with capacity=100
                        slice[0] = 1                 // assign value to index 0

                        slice := []int{1, 2, 3}      // initialized with 3 elements
                        slice = append(slice, 4)     // append value to slice
                        fmt.Println(slice)           // => [1 2 3 4]
                        fmt.Println(len(slice))      // => 4
                        fmt.Println(slice[1:2])      // => [2] (slicing)
Maps 
                    
                        // A Map is a collection of key-value pairs
                        var dict map[string]int           // empty map
                        dict := make(map[string]int)      // empty map
                        dict := map[string]int{"key": 42} // initialized

                        dict["key"] = 42         // assign value to key
                        value := dict["key"]     // getting value of key
                        value, ok := dict["key"] // getting value, check existence
                        fmt.Println(value, ok)   // => 42 true
                        delete(dict, "key")      // delete key-value pair
                        clear(dict)              // clear all key-value pairs

Language Specifics

A Handcrafted Selection

Visibility

  • visibility is determined by the upper and lower case of names
  • if a name begins with an uppercase letter, it is exported
    (accessible from other packages)
  • if a name begins with a lower-case letter, it is private
    (only visible within the same package)

                    
                        // ./user/user.go
                        package user

                        type User struct { // User is exported
                            Name   string  // exported field
                            active bool    // private field
                        }

                        // ./main.go
                        package main
                        import "user"
                        user := user.User{Name: "Alice"}
                        user.active = true // not possible
Pointers 
                    
                        var pointer *int            // default value is nil
                        fmt.Println(pointer == nil) // => true

                        value := 1
                        pointer = value  // not possible, int vs *int
                        pointer = &value // possible, pointing to the address of value
                        *pointer = value // possible, dereferencing to current value
  • no pointer arithmetics
  • no manual memory management
  • pointer parameters of functions are "pass by value"
Interfaces 
                    
                        // Stringer is an built-in interface
                        type Stringer interface {
                            String() string
                        }

                        type Session struct { /* ... */ }

                        func (s Session) String() string { /* ... */ }

                        func print(s Stringer) {
                            fmt.Println(s.String())
                        }
  • no explicit interface implementation
  • a type automatically fulfills an interface
    when it implements the required functions

How to use interfaces in Go

Errors 
                    
                        package user

                        func persist(u User) error { /* ... */ }

                        func Create(name string) (User, error) {
                            // ...
                            err := persist(u)
                            if err != nil {
                                return User{}, err
                            }
                            return u, nil
                        }
  • no exceptions, only the build-in interface error
  • it‚Äôs idiomatic to communicate errors via an explicit, separate return value
  • errors handled by using the same language constructs employed for other, non-error tasks
  • errors can be wrapped
Concurrency - Go Routines 
                    
                        func f(from string) {
                            fmt.Println(from)
                        }

                        func main() {
                            f("direct")
                            for i := range 10000 {
                                from := fmt.Sprintf("goroutine %d", i)
                                go f(from)
                            }
                            time.Sleep(3*time.Second)
                            fmt.Println("done")
                        }
                    
                
                            
                                // direct
                                // ...
                                // goroutine 9981
                                // goroutine 3953
                                // goroutine 9982
                                // ...
                                // done
  • go executes functions concurrent
  • eponym of the language name
  • simple syntax and efficient concurrency
  • very lightweight (compared to threads)
Concurrency - Channels 
                    
                        messages := make(chan string)

                        go func() { messages <- "ping" }()

                        msg := <-messages
  • channels are the typed pipes that connect concurrent goroutines
  • values can be send into channels from one goroutine and received in another goroutine
  • channel <- value syntax to send into a channel
  • <-channel syntax to receive from a channel
  • by default sends and receives block until both the sender and receiver are ready
  • channels can be unbuffered (default) and buffered
